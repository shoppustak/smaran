```typescript
interface AsrProvider {
  transcribe(audio: Bytes, opts: { languageHint: "hi-IN" }): Promise<{
    transcript: string;         // Devanagari + Latin mixed, as heard
    providerMeta?: unknown;
  }>;
}
```
Use Node's `Buffer` for the `Bytes` placeholder type.

External-API-proxy shape to mirror (`panchang.ts`, full pattern — env-gated key, fetch, `.ok`
check, normalize response):
```typescript
const VEDIKA_API_KEY = process.env.VEDIKA_API_KEY;
const VEDIKA_BASE_URL = VEDIKA_API_KEY
  ? (process.env.VEDIKA_API_BASE_URL ?? "https://api.vedika.io")
  : "https://api.vedika.io/sandbox";
...
try {
  const upstream = await fetch(`${VEDIKA_BASE_URL}/astrology/panchang`, { method: "POST", ... });
  if (!upstream.ok) {
    req.log.error({ status: upstream.status }, "Vedika Panchang API returned an error");
    res.status(502).json({ error: "..." });
    return;
  }
  ...
} catch (err) {
  req.log.error({ err }, "Failed to reach Vedika Panchang API");
  res.status(502).json({ error: "..." });
}
```
Applied here: since `asr/sarvam.ts`/`asr/openai.ts` are library functions (not routes), the
`if (!upstream.ok)` / `catch` skeleton throws/returns a typed error instead of writing an HTTP
response — same library-function-vs-route adjustment Phase 1's PATTERNS.md made for `geocoding.ts`.
